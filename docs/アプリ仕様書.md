# AIミニアプリ仕様書 v2.1
## AI文章リライトツール

---

# 1. 概要

本アプリは、ユーザーが入力した文章をAIによってリライト（書き換え）する  
シンプルな個人向け文章改善ツールである。

ユーザーは文章を入力し、用途に応じた変換モードを選択することで  
AIが適切な文章へと自動的に書き換える。

主な目的は、日常的な文章作成の効率化および文章品質の向上である。

特徴

- シンプルな単一画面UI
- ワンクリックで文章改善
- AIを意識させない軽量ツール
- 日常的に「ふと使いたくなる」設計

# 4. 機能仕様

## 4.1 文章入力

ユーザーは自由な文章を入力する。

入力形式

```
textarea
```

仕様

- 最大文字数：2000文字
- 最小文字数：5文字
- 空入力の場合は送信不可

プレースホルダー

```
ここに文章を入力してください
```

入力例

```
ミーティングの日程なんですが、
ちょっと予定が変わってしまったので
来週に変更できないでしょうか
```

---

## 4.2 リライトモード

ユーザーは4つの文章変換モードから選択できる。

### ビジネス文書

ビジネス用途に適した、適切で洗練されたビジネス文書へ書き換えます。

mode

```
polite
```

---

### 要約

意味を保ちながら、文章を端的に要約します。

mode

```
short
```

---

### 分かりやすくする

文章構造を整理し、読みやすくする。

mode

```
clear
```

---

### カジュアルにする

友人向けの自然な口語表現へ変換する。

mode

```
casual
```

---

# 5. AI処理

AIはGoogle Gemini APIを利用して文章を生成する。

使用API

Google Generative AI API

使用モデル

```
gemini-1.5-flash
```

理由

- レスポンスが高速
- 無料枠が利用可能
- 文章生成に適している

### 5.0 モデル情報の明示

ユーザーが現在どの AI モデル（Gemini, Groq 等）を利用しているか把握できるよう、画面上にモデル情報を表示する。

- 表示位置：画面右上（常時表示）
- 表示形式：バッジ状またはテキスト
- 拡張性：複数のプロバイダーやモデルが切り替わった場合でも、動的に表示内容を更新する設計とする。

---

## 5.1 処理フロー

1. フロントエンドから文章とモードを受信  
2. モードに応じてプロンプトを生成  
3. Gemini APIへリクエスト送信  
4. AI生成結果を取得  
5. フロントエンドへレスポンス返却  

---

# 15. 入力バリデーション

ユーザー入力の品質を担保し、AI処理の安定性を確保するため  
以下のバリデーションを実装する。

入力制限

最小文字数

最大文字数

2000文字

# 16. テキスト正規化

AI生成品質を安定させるため  
入力テキストに対して簡易的な前処理を行う。

処理内容

・連続改行を最大2行に制限  
・前後の空白削除  

例

```
\n\n\n\n → \n\n
```

---

# 17. レート制限

APIの過負荷および無料枠の消費を防ぐため  
簡易的なリクエスト制限を設ける。

制限

1ユーザーあたり

10リクエスト / 分

簡易実装

フロントエンド側で  
1秒クールダウンを設ける。

連続送信時

```
少し待ってから再度実行してください
```

# 22. プロンプト出力制御

AI出力を安定させるため  
プロンプトに以下の条件を追加する。

条件

・文章のみ出力  
・解説不要  
・箇条書き禁止  

プロンプト例

```

文章のみ出力してください。

文章:
{input_text}
```
