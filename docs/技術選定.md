# AI文章リライトツール 技術選定仕様（2026年版）

## フロントエンド技術構成

| 項目 | 採用技術 | 選定理由 |
|------|------------|------------|
| フレームワーク | **Next.js 15** | React Server Components・App Routerが安定。軽量でSSR対応が容易。 |
| 言語 | **TypeScript 5.7** | 強力な型推論とstrictモードにより、開発品質が大幅向上。 |
| スタイリング | **Tailwind CSS 4.0** | クラスベースデザインで開発効率向上。最新版は軽量化と自動最適化対応。 |
| 状態管理 | **Zustand + React Query** | 小規模アプリに最適な軽量構成。非同期処理もフックベースで簡潔化。 |
| UIライブラリ | **Headless UI / Radix UI** | デザインを制約せず、アクセシビリティを確保可能。 |
| HTTP通信 | **Fetch API (ネイティブ)** | Next.js Route Handlersと統合しやすく、依存削減。 |
| ビルド環境 | **Turbopack** | 開発時のリロード高速化。Vercel環境下で最適動作。 |
| デプロイ環境 | **Vercel** | Next.js公式ホスティング。無料・CDN最適化済み。 |


## バックエンド技術構成

| 項目 | 採用技術 | 選定理由 |
|------|------------|------------|
| フレームワーク | **FastAPI 0.115** | 非同期I/Oが高速、スキーマ定義が型安全。API開発に最適。 |
| 言語 | **Python 3.12** | 新しい構文・型システム（PEP 695）対応で保守性が高い。 |
| サーバ | **Uvicorn + Gunicorn** | 高速非同期Webサーバ。軽量で実績豊富。 |
| SDK | **google-generativeai 0.7.5** | Gemini 1.5 Flashモデルに正式対応。レスポンス制御が容易。 |
| 設定管理 | **pydantic-settings** | .env管理を型安全に実行。環境切替も容易。 |
| ロギング | **loguru** | 美しい構造化出力、開発・本番両方で扱いやすい。 |
| テスト | **pytest + httpx** | 非同期APIテストが簡単に書ける。Mock対応も充実。 |
| APIドキュメント | **Swagger UI / ReDoc** | FastAPI標準対応。自動生成で実装負担なし。 |


## AIモデル構成

| 項目 | 採用技術 | 選定理由 |
|------|------------|------------|
| モデル | **Gemini 1.5 Flash** | 高速・低コストで短文リライトに最適。日本語精度も高い。 |
| temperature | 0.3 | 出力の一貫性重視。ビジネス文用途に安定。 |
| top_p | 0.8 | 多様性を適度に維持しつつ自然表現を確保。 |
| max_output_tokens | 500 | 文章リライト用途に十分な長さを確保。 |
| 出力ルール | 文章のみ、解説禁止、箇条書き禁止 | 出力を安定化し、UIでそのまま利用できる。 |


## インフラ構成

| 項目 | 採用技術 | 選定理由 |
|------|------------|------------|
| コンテナ基盤 | **Docker Compose v2.28** | ローカル・本番環境を一貫した構成で再現可能。 |
| フロントホスティング | **Vercel** | デプロイ速度が速く、Next.jsとの統合性が高い。 |
| バックホスティング | **Render / Fly.io** | 無料SSL対応でAPI運用が容易。 |
| CI/CD | **GitHub Actions** | 自動テスト＋デプロイがシンプルに実行可能。 |
| 環境変数管理 | **direnv + dotenv-linter** | 環境変数の安全運用と検証が可能。 |
